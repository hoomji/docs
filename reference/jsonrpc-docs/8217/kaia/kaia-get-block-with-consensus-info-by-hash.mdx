---
title: 'kaia_getBlockWithConsensusInfoByHash'
description: 'Returns block information along with the consensus committee metadata for the specified block hash.'
---
### Parameters

  <ParamField query='chainId' type='string' default='8217' required>
    Unique identifier for a blockchain network.
  </ParamField>
  <ParamField body='id' type='number' default={1} required>
    Request identifier used to match responses.
  </ParamField>
  <ParamField body='jsonrpc' type='string' default='2.0' required>
    JSON-RPC version.
  </ParamField>
  <ParamField body='method' type='string' default='kaia_getBlockWithConsensusInfoByHash' required>
    JSON-RPC method name.
  </ParamField>

<ParamField body='params' type='object' default={[""]} placeholder='[""]' required>
  Parameters: [blockHash].
  
  Positional parameters for <code>kaia_getBlockWithConsensusInfoByHash</code>:
  <ul>
  <li><code>params[]</code>: 32 Bytes - hash of the block where this transaction was included. Null when pending.</li>
  </ul>
</ParamField>

### Returns

<ResponseField name='result' type='object'>
  Block data combined with consensus information.
  <Expandable title='properties'>
    <ResponseField name='block' type='object'>
      Block information similar to kaia_getBlockByHash.
    </ResponseField>
    <ResponseField name='committee' type='array'>
      Addresses of committee members.
      <Expandable title='items'>
        <ResponseField name='items[]' type='string'>
          The address.
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name='proposer' type='string'>
      Address of the committee member who proposed the block.
    </ResponseField>
  </Expandable>
</ResponseField>
