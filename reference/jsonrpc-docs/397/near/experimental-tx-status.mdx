---
title: 'EXPERIMENTAL_tx_status'
description: 'Queries status of a transaction by hash, returning the final transaction result and details of all receipts.'
---
### Parameters

  <ParamField query='chainId' type='string' default='397' required>
    Unique identifier for a blockchain network.
  </ParamField>
  <ParamField body='id' type='number' default={1} required>
    Request identifier used to match responses.
  </ParamField>
  <ParamField body='jsonrpc' type='string' default='2.0' required>
    JSON-RPC version.
  </ParamField>
  <ParamField body='method' type='string' default='EXPERIMENTAL_tx_status' required>
    JSON-RPC method name.
  </ParamField>

<ParamField body='params' type='object' default={{}} required>
  Checks status of a transaction by hash and includes all receipt outcomes and metadata.
  
  Parameters for <code>EXPERIMENTAL_tx_status</code>:
  <ul>
  <li><code>params.tx_hash</code>: Base58-encoded transaction hash.</li>
  <li><code>params.sender_account_id</code>: Account ID of the sender. Used to determine which shard to search in.</li>
  <li><code>params.wait_until</code>: Optional execution level to wait for before responding. Use NONE for immediate response.</li>
  </ul>
</ParamField>
