---
title: 'sui_executeTransactionBlock'
description: 'Execute the transaction and wait for results if desired. Request types: 1. WaitForEffectsCert: waits for TransactionEffectsCert and then return to client. This mode is a proxy for transaction finality. 2. WaitForLocalExecution: waits for TransactionEffectsCert and make sure the node executed the transaction locally before returning the client. The local execution makes sure this node is aware of this transaction when client fires subsequent queries. However if the node fails to execute the transaction locally in a timely manner, a bool type in the response is set to false to indicated the case. request_type is default to be WaitForEffectsCert unless options.show_events or options.show_effects is true.'
---

  <ParamField query='chainId' type='string' default='sui' required>

    JSON-RPC version.
  </ParamField>
  <ParamField body='method' type='string' default='sui_executeTransactionBlock' required>
    JSON-RPC method name.
  </ParamField>

<ParamField body='params' type='object' default={["AAACACBqEB6aOvXIBwES+Ahkizbvv43uihqC3kbZUE6WoRCKFwEAjvdvVs...",[],{},"WaitForEffectsCert"]} placeholder='["AAACACBqEB6aOvXIBwES+Ahkizbvv43uihqC3kbZUE6WoRCKFwEAjvdvVs...",[],{},"WaitForEffectsCert"]' required>
  Execute a signed transaction and optionally wait for effects or local execution.
  
  Positional parameters for <code>sui_executeTransactionBlock</code>:
  <ul>
  <li><code>params[0]</code>: Base64-encoded BCS serialized transaction data (without type tag).</li>
  <li><code>params[1]</code>: Array of base64-encoded signatures.</li>
  <li><code>params[2]</code>: Optional transaction response options.</li>
  <li><code>params[3]</code>: Execution mode indicating how long to wait for confirmation.</li>
  </ul>
</ParamField>
