### Parameters

Parameters to specify the range of containers to fetch.

- `startIndex` (integer): The index of the first container to return.
- `numToFetch` (integer): The number of containers to fetch (max 1024).
- `encoding` (string): The encoding format for the returned container bytes. Must be "hex".

### Returns

<ResponseField name='result' type='array'>
  List of container objects in sequential index order.
  <Expandable title='items'>
    <ResponseField name='items[]' type='object'>
      <Expandable title='properties'>
        <ResponseField name='id' type='string'>
          Unique ID of the container.
        </ResponseField>
        <ResponseField name='bytes' type='string'>
          Hex-encoded byte representation of the container.
        </ResponseField>
        <ResponseField name='timestamp' type='string'>
          Unix timestamp when the container was accepted.
        </ResponseField>
        <ResponseField name='encoding' type='string'>
          Encoding of the `bytes` field (always "hex").
        </ResponseField>
        <ResponseField name='index' type='string'>
          Index position of the container.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>
