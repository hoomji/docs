### Parameters

Encoding configuration for the returned data.

- `encoding` (string): Specifies the encoding format. Must be "hex".

### Returns

<ResponseField name='result' type='object'>
  Details of the last accepted container on the chain.
  <Expandable title='properties'>
    <ResponseField name='id' type='string'>
      The container ID of the last accepted container.
    </ResponseField>
    <ResponseField name='bytes' type='string'>
      Hex-encoded byte representation of the container.
    </ResponseField>
    <ResponseField name='timestamp' type='string'>
      Unix timestamp when the container was accepted.
    </ResponseField>
    <ResponseField name='encoding' type='string'>
      Encoding format used in `bytes` (always "hex").
    </ResponseField>
    <ResponseField name='index' type='string'>
      The sequential index of this container.
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>

```bash Curl
curl --request POST \
  --url https://api.uniblock.dev/uni/v1/json-rpc?chainId=43114 \
  --header 'content-type: application/json' \
  --data '{
    "id": 1,
    "jsonrpc": "2.0",
    "method": "index.getLastAccepted",
    "params": {
      "encoding": "hex"
    }
  }'
```
