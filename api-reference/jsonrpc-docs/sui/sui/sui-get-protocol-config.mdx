---
title: 'sui_getProtocolConfig'
api: 'POST https://api.uniblock.dev/uni/v1/json-rpc'
playground: 'none'
---
### Parameters

Optional array with a single protocol version to fetch configuration for.

- `params[]` (string)

### Returns

<ResponseField name='result' type='object'>
  Protocol configuration response for the given or latest protocol version.
  <Expandable title='properties'>
    <ResponseField name='attributes' type='boolean'>
      Attributes field indicates protocol configuration attributes exist.
    </ResponseField>
    <ResponseField name='featureFlags' type='boolean'>
      Whether protocol-level feature flags are supported in this version.
    </ResponseField>
    <ResponseField name='maxSupportedProtocolVersion' type='string'>
      Maximum supported protocol version for this node.
    </ResponseField>
    <ResponseField name='minSupportedProtocolVersion' type='string'>
      Minimum supported protocol version for this node.
    </ResponseField>
    <ResponseField name='protocolVersion' type='string'>
      Protocol version number being used.
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>

```bash Curl
curl --request POST \
  --url https://api.uniblock.dev/uni/v1/json-rpc?chainId=sui \
  --header 'content-type: application/json' \
  --data '{
    "id": 1,
    "jsonrpc": "2.0",
    "method": "sui_getProtocolConfig",
    "params": [
      "6"
    ]
  }'
```

</RequestExample>
<ResponseExample>

```json Success
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "attributes": true,
    "featureFlags": true,
    "maxSupportedProtocolVersion": "24",
    "minSupportedProtocolVersion": "1",
    "protocolVersion": "6"
  }
}
```

</ResponseExample>
